Part 1 — What We're Building

Hook solves a real problem every UT student faces: navigating a dozen separate systems (degree audit, scholarship portal, research board, advising) with zero intelligence connecting them. A student today has to manually search for scholarships they might qualify for, guess which courses to take next, and cold-email professors with no idea if there's a fit.

Hook replaces all of that with one profile, three AI engines, and a chat interface:

EngineWhat it doesWhy it's better than existing tools
Academic AdvisorReads your completed courses, finds gaps, tells you exactly which courses to take next and whyUT Degree Audit only shows a static checklist. Hook adds AI-driven urgency ranking and graduation risk alerts.
Financial Aid MatcherTakes your profile and surfaces every scholarship you qualify for, ranked by deadline and match strengthUT Scholarship Portal requires manual searching. Hook does it proactively, including external awards the portal doesn't list.
Research Opportunity FinderMatches your skills and interests to UT labs and programs — then drafts a personalized cold email to the professorEUREKA only shows what professors manually posted. Hook matches against all lab data and writes the email for you.

The fourth piece is a RAG chat interface — an AI assistant that knows your profile and can answer any UT campus question (study spaces, deadlines, policies) in the context of your specific situation.

The pitch in one line "EUREKA shows you what professors posted. Hook finds the labs that match your skills even if they never posted anything. The scholarship portal shows what UT offers. Hook finds the $10,000 NSF grant your portal doesn't list."

Part 2 — Tools You Need to Know

2.1 What is Codex? (Plain English)

Codex is an AI coding agent made by OpenAI. Think of it like having a very fast, very capable senior engineer sitting next to you. You describe what you want in plain English (or detailed technical specs), and it writes the code, creates files, and can even run commands in your terminal.

The key insight: Codex works best when you give it a very specific, detailed prompt. Vague prompts like "make a form" get mediocre results. Precise prompts like "make a 4-step React form using shadcn/ui Card and Input components that maps to this exact TypeScript interface" get production-quality results.

Important mental model Codex is not a search engine or a chatbot. It is an autonomous agent that reads your files, writes code, and executes terminal commands. Always review what it produces before merging. It's fast but not infallible.

2.2 What is a Git Worktree? (Plain English)

Normally, a Git repository is like a single desk where you do your work. Only one person can work on the main codebase at a time without getting in each other's way.

A worktree is like photocopying the desk and setting up identical workstations side by side. Each person gets their own copy of all the files, on their own branch, and changes don't interfere with each other. At the end, we "merge" all the desks back into one.

For this project: Person A works on main, Person B works in ../hook-academic, Person C works in ../hook-matchers. All three are working simultaneously without blocking each other.

Save the file, then git add [filename] and continue with git merge --continue

Vercel-Specific: Vector Store Cold Start

Vercel serverless functions don't keep memory between cold starts. The vector store is an in-memory singleton, so if a function "goes cold" (no traffic for a few minutes), the store empties.

The code already handles this: every API route that uses the RAG store checks isInitialized() and re-runs init if needed. But for the demo:

  • Open the app 5 minutes before demo time and click "Demo Mode" — this warms everything up
  • Send one chat message to pre-warm the RAG store
  • If you see a slow first response (>10 seconds): that's the store re-initializing. It'll be fast after that. Explain this to judges as expected behavior ("we pre-warm for production").
  • As a last resort: run npm run dev locally and demo from localhost

ProblemSymptomsFix
Codex task fails / times out Codex shows error or doesn't commit Re-run the task with the same prompt. If it fails twice, write the file manually — the prompts in this guide are detailed enough to implement by hand. Don't waste more than 10 minutes on a single stuck Codex task.
TypeScript errors after merge npm run build shows type errors Run npm run build 2>&1 | head -50 to see the first errors. Most are import path issues — check that the file exists at the path being imported. Missing property errors: check lib/types.ts and add the missing field.
OpenAI API returns 429 (rate limit) Chat or research matcher fails with 429 error The embedding cache in embeddings.ts prevents most re-calls. If you hit limits: (1) wait 60 seconds, (2) reduce the number of labs in faculty-labs.json to 10, (3) use the static fallback demo mode. Check your API usage at platform.openai.com/usage.
Vercel deployment fails Build error in Vercel logs Check the Vercel build log. Most common cause: missing environment variable (OPENAI_API_KEY not set in Vercel). Second most common: TypeScript error that npm run build locally would have caught. Fix locally first, push again.
Vector store empty on demo Chat returns "I don't have information about that" for everything Open the browser console on the demo URL. Degree analysis returns empty recommendations Academic page shows no courses Check that the profile's major exactly matches a key in the analyze.ts mapping ("Computer Science" not "CS"). Check that cs-bscs.json exists in data/degrees/. Test the API route directly with curl (see Part 8).
